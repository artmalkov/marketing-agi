# Find Competitors Rulebook

## Document Metadata

- **Version:** 1.0.0
- **Purpose:** Define rules for finding and collecting competitor information for a student project

---

## Purpose

This rulebook defines the process for finding main competitors (10+), collecting basic information (name, website, revenue, investments, etc.), and saving it to the `competitors/` folder in the student project root.

---

## Related Concepts

- **Competitor Analysis** (`knowledge/competitor-analysis/concepts/concepts/competitor-analysis.md`)
- **Product Genotype** (`knowledge/genotype/concepts/concepts/product-genotype.md`) — for understanding product context

---

## Step 1. Определение студенческого проекта (критично)

**Очень важно не ошибиться с выбором проекта.**

### Способы определения целевого проекта

1. **Явно указанный путь или источник**
2. **Открытый в IDE файл** из `student-projects/[student-name]/...`
3. **Контекст запроса** — имя студента или проекта

### Защита от ошибки

- [ ] Если проект не очевиден — спроси пользователя
- [ ] Все пути: `student-projects/[student-name]/competitors/`
- [ ] Никогда не пиши в `student-projects/` без имени студента

---

## Step 2. Определение контекста продукта

Выбери **один** из вариантов (по приоритету):

### Вариант A: Ссылка на студенческий проект

- Изучи `student-projects/[student-name]/genotype/genotype.csv` — потребности и контекст
- Изучи `student-projects/[student-name]/genotype/call-database.md` — описание продукта и потребностей
- Изучи `student-projects/[student-name]/webpages/landing.html` — позиционирование
- Сформируй краткое описание продукта для поиска конкурентов

### Вариант B: Явное описание продукта от пользователя

- Пользователь сообщил при вызове, что за продукт
- Используй это описание как контекст для поиска

### Вариант C: Список названий или ссылок конкурентов

- Пользователь передал несколько названий или URL сайтов конкурентов
- Это не поиск «с нуля», а расширение/валидация списка — найди дополнительных конкурентов в той же нише и добавь к списку

---

## Step 3. Поиск конкурентов в интернете

### 3.1. Инструменты

Используй доступные средства:
- **Web Search** — поиск по запросам вида «[тип продукта] конкуренты», «[ниша] alternatives», «[продукт] competitors»
- **mcp_web_fetch** — загрузка страниц сайтов конкурентов для проверки и сбора данных
- Рейтинги, обзоры (Crunchbase, SimilarWeb, отраслевые рейтинги — если есть доступ)

### 3.2. Целевое количество

Собери **не менее 10 основных конкурентов**. Приоритет — прямые конкуренты в той же нише.

### 3.3. Информация по каждому конкуренту

Собери по возможности:
- **Название** (обязательно)
- **Сайт / URL** (обязательно)
- **Выручка** (если доступно)
- **Привлечённые инвестиции** (если доступно)
- **Краткое описание** — чем занимаются, ключевое предложение
- **Примечания** — рейтинги, количество пользователей и т.д.

---

## Step 4. Сохранение результатов

### 4.1. Структура папок

```
student-projects/[student-name]/
  competitors/
    competitors.md           ← создаётся/обновляется
    [competitor-name]/       ← подпапка для каждого конкурента (создаётся при необходимости)
```

### 4.2. Файл competitors.md

Формат (см. `knowledge/competitor-analysis/data/templates/competitors-template.md`):

- Заголовок «Конкуренты»
- Блок «Контекст продукта» — краткое описание продукта
- Таблица: # | Название | Сайт | Выручка | Инвестиции | Примечания
- Список подпапок (если созданы)

### 4.3. Подпапки конкурентов

Для каждого конкурента создай подпапку `competitors/[competitor-name]/`. Имя папки — нормализованное название (латиница, без спецсимволов, например `competitor-name`).

Пока можно оставить папки пустыми или добавить минимальный `website.md` с выдержками с сайта, если они уже собраны. Детальные материалы — в рамках Job «Глубокий анализ конкурента».

---

## Step 5. Проверка перед сохранением

- [ ] Путь: `student-projects/[student-name]/competitors/`
- [ ] competitors.md содержит таблицу с 10+ конкурентами (где возможно)
- [ ] Название и сайт заполнены для каждого
- [ ] Подпапки созданы с нормализованными именами

---

## Output

- **competitors.md** — сводный список конкурентов
- **Подпапки** `competitors/[competitor-name]/` для последующего глубокого анализа
- **Сообщение пользователю:** сколько конкурентов найдено, какие данные собраны, где сохранено
