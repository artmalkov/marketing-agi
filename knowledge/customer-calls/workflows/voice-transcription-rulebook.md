# Voice Transcription Rulebook

Этот воркфлоу транскрибирует аудиофайлы через Nexara Speech-to-Text API и сохраняет результат рядом с исходным файлом в формате `.md`.

---

## Требования

- В окружении должна быть установлена переменная `NEXARA_API_KEY`
- Утилиты: `curl`, `bash` (стандартные на macOS/Linux, ничего устанавливать не нужно)

---

## Шаги

### 1. Определи целевые файлы

Если пользователь указал конкретный файл — используй его.

Если пользователь указал папку или написал «все файлы» — найди все аудиофайлы в директории `customer-calls/call-recordings/` нужного студенческого проекта.

Поддерживаемые форматы: `mp3`, `wav`, `m4a`, `flac`, `ogg`, `opus`, `mp4`, `mov`, `avi`, `mkv`.

Если файлов нет — сообщи пользователю и остановись.

---

### 2. Запусти транскрибацию

Выполни команду через Bash для каждого аудиофайла (или всей папки сразу):

**Один файл:**
```bash
export NEXARA_API_KEY="ваш_ключ"
bash scripts/transcribe.sh "путь/к/файлу.mp3"
```

**Вся папка:**
```bash
export NEXARA_API_KEY="ваш_ключ"
bash scripts/transcribe.sh --dir "student-projects/[имя]/customer-calls/call-recordings"
```

**С диаризацией спикеров** (если нужно разделить реплики по спикерам, требует `jq`):
```bash
bash scripts/transcribe.sh --diarize "путь/к/файлу.mp3"
```

**С указанием языка** (опционально, улучшает точность):
```bash
bash scripts/transcribe.sh --lang ru "путь/к/файлу.mp3"
```

> Скрипт автоматически сохранит `.md`-файл рядом с аудиофайлом, с тем же именем.
> Например: `звонок.mp3` → `звонок.md`

---

### 3. Проверь результат

После выполнения убедись, что:
- `.md`-файл создан рядом с исходным аудиофайлом
- Файл не пустой и содержит осмысленный текст
- Язык транскрипции соответствует языку аудио

Если файл пустой или API вернул ошибку — сообщи пользователю текст ошибки и остановись.

---

### 4. Сообщи пользователю

По завершении кратко сообщи:
- Сколько файлов было транскрибировано
- Пути к созданным `.md`-файлам
- Если были ошибки — какие именно

---

## Примечания

- `NEXARA_API_KEY` — если ключ не установлен глобально, попроси пользователя его предоставить.
- Диаризация (разделение по спикерам) занимает дольше и стоит вдвое дороже обычной транскрипции. Используй только по запросу.
- Файлы, у которых уже есть соответствующий `.md`-файл, скрипт перезапишет без предупреждения — уточни у пользователя, если это критично.
